NT MINIPORT SHIM FOR WIN9X - BUILD INSTRUCTIONS
================================================

WHAT THIS IS:

A Win9x VxD/PDR that loads an unmodified NT4 SCSI miniport driver
(.sys) by providing a compatibility shim for the 22 ScsiPort API
functions. No known prior implementation exists in the public record.


FILE INVENTORY:

  NTMINI.C    - Main source: ScsiPort shim + PE loader + IOS bridge
  BUILD.TXT   - This file


PREREQUISITES:

One of these development environments:

A) Windows 98 DDK (Microsoft, circa 1998)
   - Includes VxD headers (vmm.h, vxdwraps.h, ios.h, etc.)
   - Includes MASM 6.11d for VxD assembly boilerplate
   - Can target .VXD or .PDR output

B) Open Watcom C/C++ (free, open source)
   - Download from https://github.com/open-watcom/open-watcom-v2
   - Supports VxD targets with -bt=windows flag
   - Needs VxD headers (from Win98 DDK or community headers)

C) MASM 6.11 + Microsoft C/C++ (for purists)
   - Classic Win9x driver development setup
   - VxD boilerplate in .ASM, logic in .C


BUILD STEPS:

These are approximate -- adjust for your toolchain.

1. Install Win98 DDK or Open Watcom.

2. Set up include paths to VxD headers:
   - VMM.H (Virtual Machine Manager)
   - VPICD.H (Virtual PIC Device, for interrupts)
   - IOS.H (I/O Supervisor, for port driver registration)
   - IOR.H (I/O Request structures)
   - BLOCKDEV.H (Block device interfaces)

3. Write VxD assembly wrapper (NTMINI.ASM):
   - Device Descriptor Block (DDB) declaring port driver
   - Control procedure handling Sys_Dynamic_Device_Init/Exit
   - IRQ handler trampoline

4. Compile:
   cl -c -Gs -G3 -Zl NTMINI.C
   ml -c -coff NTMINI.ASM
   link -vxd NTMINI.OBJ NTMINIASM.OBJ -def:NTMINI.DEF

   (Exact flags vary by toolchain version)

5. Output: NTMINI.PDR (or NTMINI.VXD)

6. Copy to Win98 machine:
   C:\WINDOWS\SYSTEM\IOSUBSYS\NTMINI.PDR

7. Also copy NT4's atapi.sys to the same directory.

8. Install via registry or .INF file.


WHAT STILL NEEDS IMPLEMENTATION:

The C source is a working skeleton with real I/O code and
detailed pseudocode for the OS-integration parts. The parts
that need Win98 DDK headers to complete:

1. Memory allocation (HeapAllocate / _PageAllocate)
2. PE file loading (IFSMgr_Ring0_OpenCreateFile or similar)
3. IOS port driver registration (AEP handler)
4. IOR-to-SRB translation (the IOS bridge)
5. Interrupt hooking (VPICD_Virtualize_IRQ)
6. VxD assembly boilerplate (DDB, control proc)

The ScsiPort API implementations (Part 2 of NTMINI.C) are
COMPLETE and should work as-is. The port I/O functions are
just IN/OUT instruction wrappers. The timing function uses
port 0x80 reads for microsecond-level delays.


TESTING:

Phase 1: Verify PE loads without crash
  - Add debug output (VxD OutputDebugString or INT 3 breakpoints)
  - Load in Win98 VM with SoftICE or WDEB386 debugger
  - Confirm atapi.sys's DriverEntry is called
  - Confirm ScsiPortInitialize receives valid HW_INITIALIZATION_DATA

Phase 2: Verify hardware detection
  - Confirm HwFindAdapter succeeds
  - Confirm HwInitialize succeeds
  - Check that the miniport reads the correct I/O ports
  - Verify IDENTIFY PACKET DEVICE data is read

Phase 3: Verify I/O
  - Submit a test SRB (INQUIRY command)
  - Verify the miniport completes it
  - Check that IOS sees the device

Phase 4: Integration
  - Full IOS bridge with IOR queuing
  - CD-ROM file system (CDFS) should auto-mount
  - Test reading files from DVD
